## Usage

### Quick Start (5 minutes)

```bash
# 1. Install dependencies
sudo apt install ros-humble-moveit-task-constructor-*
git clone https://github.com/xArm-Developer/xarm_ros2.git ~/xarm_ws/src/
cd ~/xarm_ws && colcon build

# 2. Build this workspace
cd RSS_Workshop
colcon build --symlink-install
source install/setup.bash

# 3. Install Python dependencies
cd agent
pip install -r simple_requirements.txt
echo "ANTHROPIC_API_KEY=sk-ant-xxx" > .env

# 4. Launch robot (in another terminal)
ros2 launch xarm_moveit_config xarm_moveit_fake.launch.py dof:=6 robot_type:=xarm

# 5. Run demo
./scripts/run_demo.sh
```

### Example Interaction

```
You: What objects are in the scene?
Agent: There are 2 objects: object_1 and object_2

You: Pick up object_1
Agent: ✅ Successfully picked up object_1

You: Pour it into object_2
Agent: Executing pour sequence...
        1. Moving to pour position... ✅
        2. Tilting to pour... ✅
        3. Placing back... ✅
        Task complete!
```

## Design Decisions

### Why This Structure?

1. **Minimal but Complete**: Only essential files for a working demo
2. **Clear Separation**: Agent (Python) vs. ROS2 (C++/Python) vs. Docs
3. **Easy to Understand**: Flat structure, descriptive names
4. **Reproducible**: All dependencies documented, build tested

### Why Exclude XYZ?

- **External packages**: Too large, better maintained upstream
- **Training data**: User-specific, regeneratable
- **Experimental code**: Not ready for publication
- **Build artifacts**: Should be regenerated

### Why Include MCP Server?

Although the README mentions "removing MCP", the `mtc_mcp_server.py` is kept for:
- Reference implementation of MCP protocol
- Comparison with direct action library approach
- Potential future use

It's not used by default in the agent.

## Known Limitations

### Current Constraints

1. **No Object Detection Models**: Users must set up their own detection or use manual object IDs
2. **No Calibration Data**: Hand-eye calibration must be done by each user
3. **Simulation Limited**: Full MTC execution may have issues in fake controllers
4. **Single Robot**: No multi-robot support

### Workarounds

1. **No detection**: Use manual object IDs (e.g., "pick object_1")
2. **No calibration**: Run in simulation with known transforms
3. **Simulation issues**: Use dry-run mode (`--dry-run` flag)
4. **Multi-robot**: Modify code to add robot_id parameter
